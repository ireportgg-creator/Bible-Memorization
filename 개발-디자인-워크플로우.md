# 앱 개발 + 디자인 통합 워크플로우

> Bible Memorization 앱 개발 경험을 바탕으로 정리한 효율적인 개발-디자인 프로세스

---

## 핵심 흐름

```
아이디어
   ↓
Claude Code — 기술 플랜 수립
   ↓
Claude Design — 목업 + 프로토타입
   ↓
Claude Code — 실제 구현
   ↓
테스트 → 출시
```

---

## 단계별 설명

### 1단계: Claude Code에게 플랜 요청

앱 아이디어를 설명하고 다음을 요청한다.

- 필요한 화면 목록
- 화면 간 네비게이션 흐름
- 데이터 모델 (어떤 정보를 저장할지)
- 핵심 기능 목록

**팁:** *"Claude Design 프롬프트로 쓸 수 있게 정리해줘"* 라고 추가하면
디자인 도구에 바로 붙여넣을 수 있는 형태로 받을 수 있다.

---

### 2단계: Claude Design에서 목업 + 프로토타입 작성

Claude Code에서 받은 플랜을 프롬프트로 넘겨 다음을 제작한다.

**목업(Mockup)**
- 앱의 시각적 디자인 (색상, 폰트, 레이아웃)
- 각 화면의 구체적인 모양

**프로토타입(Prototype)**
- 목업에 실제 동작을 더한 것
- 버튼을 탭하면 다음 화면으로 전환되는 인터랙션 포함
- 실제 데이터는 없고 가짜 데이터로 작동

---

### 3단계: Claude Design에서 스펙 추출

디자인이 확정되면 구현에 바로 쓸 수 있는 스펙을 요청한다.

**요청 프롬프트 예시:**

```
이 디자인의 SwiftUI 구현 스펙을 뽑아줘:
- Color extension (hex 값 포함)
- Font 스타일 (size, weight)
- 카드 컴포넌트 (corner radius, shadow, padding)
- 버튼 스타일
AppTheme.swift 파일 형태로 줘.
```

```
HOME 화면을 SwiftUI View 구조로 분해해줘.
컴포넌트 계층과 각 컴포넌트에 필요한 데이터를 정리해줘.
```

```
이 앱의 화면 전환 흐름을 정리해줘.
sheet / fullScreenCover / NavigationLink 중 어떤 걸 쓸지도 포함해서.
```

---

### 4단계: Claude Code로 구현

디자인 스펙 + 스크린샷을 Claude Code에 넘겨 구현을 요청한다.
처음부터 확정된 디자인 기준으로 만들기 때문에 나중에 전체를 뒤엎는 일이 없다.

---

## 각 도구의 역할

| 도구 | 역할 | 잘하는 것 |
|---|---|---|
| Claude Code | 기술 설계 + 구현 | 데이터 구조, 코드 작성, 빌드 |
| Claude Design | 시각 설계 | 목업, 프로토타입, 색상/레이아웃 |

---

## 언어/기술 선택 기준

개발을 시작하기 전 **기획 단계**에서 결정한다.

### 주요 기준

1. **플랫폼** — 어디서 실행되는가
   - iPhone/iPad → Swift (SwiftUI)
   - Android → Kotlin
   - 웹 브라우저 → JavaScript (React 등)
   - iOS + Android 동시 → Flutter, React Native
   - 서버/백엔드 → Python, Node.js, Go 등

2. **성능** — 게임, 카메라, AR처럼 성능이 중요하면 네이티브 언어 선택

3. **개발자 역량** — 현실적으로 가장 큰 변수

> "어느 기기에서 실행되느냐"가 언어를 거의 결정한다.

## 이번 프로젝트에서 배운 교훈

### ❌ 비효율적인 순서 (이번에 한 방식)
```
기능 구현 → 디자인 생성 → 전체 리팩터 (이중 작업 발생)
```

### ✅ 효율적인 순서
```
Claude Code로 플랜 수립
→ Claude Design으로 디자인 확정
→ 스펙 추출
→ Claude Code로 처음부터 올바른 디자인으로 구현
```

---

## 스펙 추출 한 줄 요약

Claude Design 채팅창에 이 한 문장만 입력하면 된다:

> **"이 디자인을 SwiftUI로 구현할 수 있도록 모든 스펙을 정리해줘"**

색상, 폰트, 컴포넌트 구조, 화면 흐름까지 한 번에 뽑아준다.
그걸 그대로 Claude Code에 붙여넣으면 구현 시작 가능.

---

## 용어 정리

| 용어 | 설명 |
|---|---|
| 목업(Mockup) | 앱의 시각적 모양만 표현한 정적 화면 |
| 프로토타입(Prototype) | 목업에 실제 동작(클릭, 화면 전환)을 추가한 것 |
| 디자인 스펙 | 색상 hex 값, 폰트 크기, 여백 등 구현에 필요한 수치 |
| 네이티브 앱 | 각 플랫폼 전용 언어로 만든 앱 (Swift, Kotlin 등) |
| 크로스플랫폼 | 하나의 코드로 여러 플랫폼에서 실행되는 앱 (Flutter 등) |
